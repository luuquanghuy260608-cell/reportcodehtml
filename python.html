Python Study Guide

Python is a high-level programming language. It is easy to read and is often used for web development, data science, automation, and machine learning.

1. What It Is

Beginner-friendly idea
  • Python is beginner-friendly and easy to read.
  • It supports many programming styles, including procedural, object-oriented, and functional programming.
  • It is widely used in AI, data science, web development, and automation.

Example: A simple Python program:

name = "Huy"
print("Hello", name)

2. Key Concepts

  • Variables store data, such as numbers or text.
  • Data types include integers, floats, strings, and booleans.
  • Conditionals use if, elif, and else.
  • Loops help repeat actions with for and while.
  • Functions make code reusable.
  • Lists and dictionaries help organize data.

Example:

age = 20
if age >= 18:
    print("Adult")
else:
    print("Teen")

3. Key Code Example

def greet(name):
    return f"Hello, {name}" 

print(greet("Alice"))

4. Recursion

Important concept

Recursion is a technique where a function calls itself to solve a smaller version of the same problem. It is useful for tasks such as factorial calculation, Fibonacci numbers, tree traversal, and searching.

  • A recursive function needs a base case to stop calling itself.
  • Each recursive call should move closer to the base case.
  • If the base case is missing or the problem does not get smaller, the function can run forever.
def factorial(n):
    if n == 0:
        return 1
    return n * factorial(n - 1)

print(factorial(5))

5. Example: Fibonacci

def fibonacci(n):
    if n <= 1:
        return n
    return fibonacci(n - 1) + fibonacci(n - 2)

print(fibonacci(6))

6. References in Python

In Python, variables do not store the actual object directly in the same way as C. They store references to objects. This means two variables can point to the same object.

  • A reference is a way to access an object.
  • When you assign one variable to another, both can refer to the same object.
  • Changing the object through one variable can affect the other.
a = [1, 2, 3]
b = a
b.append(4)

print(a)  # [1, 2, 3, 4]
print(b)  # [1, 2, 3, 4]

7. Applications of References in Python

  • Used in lists, dictionaries, and objects.
  • Useful for passing large data structures to functions without copying them.
  • Helpful in building graphs, trees, and linked structures.
  • Important when you want multiple names to refer to the same data.

Example: Modifying one list can affect another variable that points to the same object.

nums = [1, 2, 3]
other = nums
other.append(4)
print(nums)

8. List, Set, and Tuple

Core Python data types

Python has several common data structures that help store and organize values.

  • List: ordered, changeable, allows duplicates.
  • Tuple: ordered, unchangeable, allows duplicates.
  • Set: unordered, no duplicates.
numbers = [1, 2, 2, 3]      # list
coords = (10, 20)          # tuple
unique = {1, 2, 3, 3}      # set

print(numbers)
print(coords)
print(unique)

9. Functions in Python

Reusable logic

A function is a reusable block of code. It helps organize programs and avoid repetition.

  • Functions can take parameters.
  • Functions can return a value.
  • They make code easier to read and maintain.
def add(a, b):
    return a + b

print(add(3, 5))

Example: A function can also print a message without returning anything.

def greet(name):
    print("Hello", name)

greet("Alice")

10. Dictionary and Loops

A dictionary stores data as key-value pairs. Loops help you repeat actions over collections.

  • Dictionary: uses keys and values.
  • for loop: repeats over items in a list or range.
  • while loop: repeats while a condition is true.
student = {"name": "Alice", "age": 20}
for key in student:
    print(key, student[key])

Example: Use a loop to print numbers from 1 to 5.

for i in range(1, 6):
    print(i)

11. Important Notes

  • Recursion is elegant but can use more memory than loops.
  • Common recursive problems include factorial, Fibonacci, binary search, and tree traversal.
  • Always check the base case first.
  • Use loops when performance matters more than simplicity.
